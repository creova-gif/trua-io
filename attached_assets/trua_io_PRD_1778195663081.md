## 1. Product Overview

**Product Name:** Trua IO  
**Tagline:** AI-powered outreach. Autonomous. Personalized. Always on.  
**Type:** SaaS web application + conversational bot  
**Primary Market:** Tanzania (Dar es Salaam → Arusha → Mwanza)  
**Target Launch:** Day 7 of 7-day sprint

### Mission
Trua IO is a 24/7 autonomous AI marketing, sales, and customer service assistant that drafts and sends personalized outreach emails to Tanzanian businesses — from SMEs to large corporations and government bodies — without requiring human intervention during normal operation.

## 4. Core Architecture

```
User → Next.js Frontend (Vercel)
          ↓
     Supabase (Auth + DB + Edge Functions)
          ↓
     Claude API (AI Engine)
          ↓
     Resend (Email Infrastructure)
          ↓
     Target: Tanzanian Business Contacts
```

**API-first design:** All AI engine functionality exposed via internal REST API so CRM integrations (Salesforce, HubSpot, Zoho) can be bolted on post-launch without rewriting core logic.

## 5. MVP Feature Set (Day 7 Scope)

### 5.1 AI Email Drafting Engine
- Claude Sonnet 4 generates personalized outreach emails per contact
- Personalization variables: company name, industry, role, pain points
- Tone adaptation: professional/formal for government, warmer for SMEs
- Bilingual output: English + Swahili toggle per campaign
- Template library: 5 pre-built Tanzanian market templates at launch

### 5.2 Personalization Engine
- Company-level personalization (name, industry, city, size)
- Industry-specific value propositions (logistics, tourism, finance, retail, NGO)
- Role-based salutation logic
- City-specific reference points (Dar es Salaam, Arusha, Mwanza)

### 5.3 Autonomous 24/7 Sending Scheduler
- Cron-based scheduler (Supabase Edge Function) runs every 15 minutes
- Timezone-aware sending (EAT — East Africa Time, UTC+3)
- Optimal send-time logic: weekdays 8am–5pm EAT, skip public holidays
- Rate limiting: max 200 emails/hour per workspace (email warmup compliance)
- Retry logic: 3 attempts with exponential backoff on failure
- Dead-letter queue for permanently failed sends

### 5.4 Bot / Chat Interface
- Conversational UI embedded in dashboard sidebar
- Commands: "draft a campaign for logistics companies in Dar", "show me open rates", "pause all sending", "generate 10 new contacts from BRELA"
- Real-time streaming responses via Claude API
- Bot remembers campaign context within a session

### 5.5 Contact Management
- CSV import with field mapping UI
- Manual contact entry form
- AI web enrichment: Claude extracts company data from public web sources
- BRELA registry integration stub (scraping layer, documented for expansion)
- LinkedIn/Apollo import via CSV export (no direct API in MVP)
- Deduplication on email + phone
- Contact stage: cold → contacted → replied → qualified → converted → unsubscribed

### 5.6 Campaign Analytics Dashboard
- Sent / delivered / opened / clicked / replied / bounced / unsubscribed counts
- Per-campaign open rate, reply rate, unsubscribe rate
- Time-series chart: sends over 7-day rolling window
- Top-performing subject lines
- Contact stage funnel visualization
- Email deliverability health score (0–100)
- Export: CSV report download

### 5.7 User Auth & Roles
- Supabase Auth: email/password + magic link
- Roles: Admin, Sales User, Viewer
  - Admin: full access, billing, team management, API keys
  - Sales User: create/run campaigns, view own analytics
  - Viewer: read-only dashboard access

### 5.8 Unsubscribe & Compliance Flow
- One-click unsubscribe link in every email (required)
- Unsubscribe landing page (Trua IO branded)
- Opt-out immediately recorded in DB — no further sends to that contact
- Unsubscribe reason capture (optional dropdown)
- Physical address in email footer (CAN-SPAM)
- Privacy policy link in footer
- Data retention notice per Tanzania PDPA 2022

## 9. Compliance Requirements

### Tanzania Cybercrimes Act 2015
- No unauthorized access to computer systems for data collection
- All web scraping limited to publicly available data only
- Data breach notification procedure documented

### Tanzania Data Protection Act 2022 (PDPA)
- Privacy notice displayed on signup and in every email
- Data subject rights: access, correction, deletion (request form in-app)
- Data controller registration (TCRA) — documentation prepared
- Data processing records maintained
- No transfer of Tanzanian personal data outside EAC without consent

### CAN-SPAM Compliance
- Physical mailing address in every email footer
- Clear "From" name and email address (no spoofing)
- Honest subject lines (no deceptive headers)
- Immediate opt-out processing (within 10 business days per law, implemented same-day)

### GDPR (EU users)
- Cookie consent banner
- Lawful basis for processing documented
- DPA available on request
